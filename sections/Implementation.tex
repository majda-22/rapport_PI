\section{Environnement Technique et Choix Technologiques}

La conception de FinPulse AI repose sur une  architecture dans laquelle chaque couche technologique est sélectionnée en fonction de sa capacité à répondre précisément aux exigences fonctionnelles et non fonctionnelles de la couche concernée. Cette section détaille les choix effectués pour chacun des cinq environnements constitutifs du système. 

\subsection{Environnement Frontend}

\textbf{Angular 19}
\vspace{0.2cm}

L'interface utilisateur de FinPulse AI est développée en Angular 19. Ce choix s'est imposé pour plusieurs raisons structurantes. Angular offre un cadre applicatif complet — routing, gestion des formulaires, injection de dépendances, modularité — qui convient particulièrement à une application de la complexité de FinPulse AI, laquelle compte plus d'une dizaine de pages fonctionnelles interconnectées. Par ailleurs, Angular 19 introduit le mécanisme des Signals, une primitive réactive qui permet de gérer l'état de l'interface de manière fine et performante, en limitant les re-rendus inutiles. Dans FinPulse AI, les Signals sont utilisés de manière systématique pour gérer les états locaux des composants — données de la watchlist, scores NCI en temps réel, état des sessions du chatbot — ce qui garantit une réactivité optimale même sous forte charge de données.

\subsection{Environnement Backend et Orchestration IA}

\textbf{Spring Boot 4.0.6}
\vspace{0.2cm}

Le backend de FinPulse AI est développé avec Spring Boot, le framework Java de référence pour le développement d'APIs RESTful d'entreprise. Spring Boot joue ici un rôle central d'orchestrateur : il expose l'ensemble des endpoints REST consommés par le frontend, sécurise les accès utilisateurs, gère la logique métier, et coordonne les appels vers les différents services internes — pipeline IA Python, moteur de recommandations, système de notifications. 

La sécurité des APIs est assurée par Spring Security, configuré en mode resource server OAuth2/OIDC. Chaque requête entrante est interceptée et le JWT Keycloak associé est validé cryptographiquement. Les autorisations sont ensuite dérivées des rôles Keycloak extraits du token, et appliquées au niveau des endpoints. Cette architecture garantit qu'aucune donnée sensible ne peut être accédée sans authentification valide, et ce même dans le cas des connexions SSE, pour lesquelles une solution spécifique a été développée : le token JWT est transmis en paramètre de requête et validé par un filtre Spring dédié. 

La persistance des données relationnelles est gérée par Spring Data JPA, une couche d'abstraction ORM qui simplifie considérablement les interactions avec PostgreSQL.

\subsection{Persistance des Données et Stockage Vectoriel}

\textbf{PostgreSQL}
\vspace{0.2cm}

L'ensemble des données persistantes de FinPulse AI est stocké dans PostgreSQL, un système de gestion de bases de données relationnelles open-source de référence en termes de robustesse, de performance et de richesse fonctionnelle. Le choix de PostgreSQL s'est imposé naturellement pour plusieurs raisons : sa compatibilité avec Spring Data JPA et SQLAlchemy, sa gestion native des types de données complexes (JSONB, arrays), et surtout sa capacité à être étendu par PGVector. 

\begin{figure}[H]
    \centering
    \includegraphics[width=0.75\textwidth]{images/logo_postgre.png}
    \caption{Logo PostgreSQL}
    \label{fig:logo_postgre}
\end{figure}

\vspace{0.6cm}
\textbf{PGVector}
\vspace{0.2cm}

PGVector est une extension PostgreSQL qui permet de stocker des vecteurs à haute dimensionnalité directement dans des colonnes relationnelles ordinaires, et d'exécuter des recherches de similarité vectorielle (cosine similarity, produit scalaire) via des index HNSW (Hierarchical Navigable Small World). Dans FinPulse AI, PGVector joue un rôle central dans le système RAG (Retrieval-Augmented Generation) du chatbot : les embeddings des documents financiers (sections de filings SEC, transcriptions d'earnings calls) sont stockés dans PostgreSQL via PGVector, et lors d'une requête utilisateur, les passages les plus semantiquement pertinents sont retrouvés par recherche vectorielle avant d'être transmis au LLM comme contexte de génération. 

\vspace{0.6cm}
\textbf{Redis}
\vspace{0.2cm}

Redis complète l'infrastructure de persistance en jouant un rôle triple et central dans l'architecture : il est utilisé comme cache distribué via Spring Cache pour accélérer les lectures répétées de données peu volatiles, comme broker de messages pour Celery afin d'orchestrer les tâches d'ingestion asynchrones, et comme store de session pour les données temporaires à accès rapide. La combinaison PostgreSQL/PGVector/Redis offre ainsi à FinPulse AI une infrastructure de données à la fois cohérente (PostgreSQL), intelligente (PGVector) et performante (Redis).

\section{Étapes de Réalisation du Système}

\subsection{Présentation Générale}

Le développement de FinPulse AI a suivi une approche incrémentale.Le développement de FinPulse AI a suivi une approche incrémentale, partant d'une infrastructure de base — authentification, accès aux données, pipeline NCI minimal — pour progressivement intégrer les fonctionnalités avancées : chatbot RAG, génération de stratégies, alertes intelligentes, visualisations financières. Cette progression a permis de valider chaque composant de manière isolée avant de l'intégrer dans le système global. 

La première phase a consisté à mettre en place l'infrastructure fondamentale : le schéma de base de données PostgreSQL, la configuration du realm Keycloak, le squelette de l'application Spring Boot avec ses couches de sécurité, et l'application Angular avec son système de routing et d'authentification. La deuxième phase a introduit le pipeline IA Python, l'ingestion des données SEC, et le calcul initial du NCI. La troisième phase a développé les fonctionnalités métier avancées — chatbot, stratégies, watchlist temps réel — tandis que la quatrième phase a affiné l'expérience utilisateur, optimisé les performances, et produit la version finale de la plateforme.

\subsubsection{1.3.3.9 Notifications}

\section{Conclusion}

Ce chapitre a présenté de manière exhaustive la réalisation concrète de FinPulse AI, en parcourant successivement les choix technologiques qui structurent chaque couche du système et les interfaces utilisateurs qui en matérialisent la valeur fonctionnelle. 

La stack technologique de FinPulse AI  n'est pas le résultat de choix arbitraires, mais d'une réflexion rigoureuse sur les exigences spécifiques d'une plateforme fintech de niveau professionnel. Chaque technologie répond à un besoin précis et s'intègre harmonieusement dans l'écosystème global. 

Sur le plan des interfaces, FinPulse AI combine analyse financière avancée et expérience utilisateur fluide. 
Le design system basé sur CSS variables, le temps réel via Angular Signals et SSE, ainsi que l’agent IA RAG enrichi, assurent cohérence, réactivité et richesse fonctionnelle. 
L’ensemble positionne la plateforme comme une solution fintech moderne et professionnelle.
